           Git主线的基本操作
   git init // git 项目初始化 
   git status // 查看文件状态（保存或没保存） 
   git add . // 添加所有文件 
   git add 文件名 // 添加单个文件 
   git commit -m "描述信息" // 提交到本地仓库
   git restore 文件名 // 还原到上次提交之前 
   git log // 日志信息 获取版本号 
   git log 只能查看本次提交之前的版本 有局限性 
   git reset -- hard 版本号 // 还原到某个版本 
   git reflog // 比log更强大的日志记录 可用于版本反复横跳
   git diff // 查看所有文件的修改记录
           github远程服务器相关操作
   ssh-keygen -t rsa -C "邮箱地址" // 获取秘钥
   需要关联远程仓库才能进行操作
   git remote add origin 仓库地址
   git push origin 分支 // 把文件推入仓库
   但是这里会出问题 第一次上传会被拒绝访问
   原因是 上传文件之后会把创建仓库时自带的文件覆盖了
   解决办法：把那个文件先拉到本地 再一起上传！
   git pull origin 分支
   这时候又有问题了 也被拒绝了！原因是 之前没有访问过，所以不给你拿！
   这就很蛋疼 就像第一次找工作需要工作经验一样 没工作哪来的经验！
   解决办法： 活用百度！git 指令系统很庞大 非专业人士不可能去研究这么深入 我们也只是用一下常用的那些指令！
   git pull origin 分支 --allow-unrelated-histories // 解除访问限制
   git clone 项目地址 // 下载别人的开源代码
           Git分支的基本操作
   git branch // 查看当前分支情况 
   git branch 分支名 // 新建一个分支 
   git checkout 分支名 // 切换分支 
   为什么要用新分支 假如我需要修改这些代码的bug
   但是担心直接在主分支里改崩了 那就放到bug分支里去改
   这样，就算改崩了也不影响主分支的代码
   新建分支的目的就是为了合并分支 不能合并 分支就没有意义了
   git checkout a分支
   git merge b分支 // 把b分支合并到a
   而且各分支的提交记录也互不干扰！
   git branch -d 分支名 // 删除分支
   git tag v1.01 // 创建分支版本号 可替代很长的hash值
   git tag v1.00 某个版本hash值// 给指定版本添加版本号
   git show 创建的版本号 // 查看指定版本的diff信息
   git tag -a 版本号 -m "日志描述信息" 日志hash值 // 为某个版本日志增加描述信息
   git tag -d 版本号 // 删除某个版本
   git log --graph // 让log 信息更美观一点
